Backbone.Events

Key feature of Backbone

Bind callbacks and dispatch events

Included in:
Model, Collection, View, History

Backbone.Events

on

Bind a callback to an object

off

Unbind a callback

trigger

Dispatch an event

Events

Code example:


model.on( 'myevent', this.hello, this );

this.hello = function( obj ) {
	alert( obj.msg );
}



model.trigger( 'myevent', {msg: 'hi'} );

Models

What do they do?

  • Store data
  • Fire events
  • Provide get()/set()

Models

Code example:


var User = Backbone.Model.extend();

var fred = new User({
  name: 'Fred'
});

alert( fred.get( 'name' ) );

Models

Change event


var fred = new User({
  name: 'Fred'
});

fred.set( 'name', 'Tim' );

Fires change and change:name events

Collections

What do they do?

  • Store Models
  • Fire events
  • Provide add()/remove(), etc
  • Sync with storage

Collections

Fires add,remove,reset events.

Implements underscore list helpers

map,sort, etc...

Collections

Code example:


var User = Backbone.Model.extend();
var fred = new User({
  name: 'Fred'
});

var Users = Backbone.Collection.extend();
var userlist = new Users();

userlist.add( fred );

Views

What do they do?

  • Owns a DOM element
  • Knows about a Model/Collection
  • Handles DOM events/input
  • Binds to Model events

Views

Rendering

Render using HTML templating

Model: { name: "Fred" }

Template: <p>{{name}}</p>

Output: <p>Fred</p>

Views

Create a View:


var Panel = Backbone.View.extend({
  events: {
    'click p': 'close'
  },
  render: function() {
    this.$el.html('<p>panel</p>');
  },
  close: function() {
    this.$el.hide();
  }
});

Router

Router Setup


App.Router = Backbone.Router.extend({
  routes: {
    'user/:id': 'getUser'
  },
  getUser: function( id ) {
    App.MainView.displayUser( id );

});
